1. The Cloud Age vs. The Iron Age

(Source: Infrastructure as Code, Chapter 1, Table 1-2. Ways of working in the Cloud Age by Kief Morris.)

Feature Iron Age Cloud Age
Cost of Change High Low
View on Changes Represents failure (must be "managed," "controlled") Represents learning and improvement
Strategy Reduce opportunities to fail Maximize speed of improvement
Delivery Large batches, test at the end Small changes, test continuously
Cycles Long release cycles Short release cycles
Architecture Monolithic (fewer, larger moving parts) Microservices (more, smaller parts)
Configuration GUI-driven or physical configuration Configuration as Code

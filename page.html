main.go — Key Parts

The server reads the PORT environment variable set by Render at runtime, serves page.html via Go's html/template, and exposes a /health endpoint.

package main

import (
    "html/template"
    "log"
    "net/http"
    "os"
)

func main() {
    tmpl := template.Must(template.ParseFiles("page.html"))

    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        tmpl.Execute(w, nil)
    })

    port := os.Getenv("PORT") // Render sets this automatically
    if port == "" { port = "8080" }

    log.Printf("Listening on :%s", port)
    http.ListenAndServe(":"+port, nil)
}
// 04 — deployment steps

How I Deployed to Render.com

  1. 01

    Created a GitHub repository and pushed all project files (main.go, go.mod, page.html, render.yaml).

  2. 02

    Logged in to render.com → clicked New + → Web Service → connected the GitHub repo.

  3. 03

    Render auto-detected render.yaml. Confirmed the build command:
    go build -tags netgo -ldflags '-s -w' -o app

  4. 04

    Selected the Free plan and clicked Create Web Service. Build took ~40 seconds.

  5. 05

    Render assigned a public URL (*.onrender.com). Opened it in the browser — the page loaded successfully.

# render.yaml (infrastructure as code)
services:
  - type: web
    name: go
    runtime: go
    plan: free
    buildCommand: go build -tags netgo -ldflags '-s -w' -o app
    startCommand: ./app
Service live — render.com free tier
// 05 — reflections

What I Learned

PaaS Concept

Render abstracts away server management — just push code and the platform handles build, deploy, TLS, and routing.

render.yaml

Infrastructure-as-code file that defines the service type, runtime, and commands. Keeps config in version control.

Go Build Flags

-tags netgo produces a fully static binary. -s -w strips debug symbols — smaller, faster cold starts.

PORT Env Var

PaaS platforms inject the port at runtime. Always read os.Getenv("PORT") — never hardcode it.

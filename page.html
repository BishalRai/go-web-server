// 01 — about

What This Is

A Go HTTP web server deployed to Render.com as the Week 1 assignment for Cloud Services at OAMK. The app is a fork of the official render-examples/go-web-server quickstart, extended with a custom page and deploy config.

The server uses Go's standard library only — no frameworks, no external dependencies. It serves this HTML page via html/template and is compiled to a fully static binary before deploy.

// 04 — changes from upstream

What Was Modified in the Fork

The upstream quickstart ships with a minimal placeholder page. The following changes were made:

  • 01

    Replaced the default page.html with this custom learning diary page for the Week 1 assignment. Custom design — IBM Plex Mono + Space Grotesk, dark theme, animated grid background

  • 02

    Added a /health endpoint in main.go for Render's uptime checks. Returns HTTP 200 OK — useful for monitoring and health probes

  • 03

    Added render.yaml to pin build & start commands explicitly in version control. Infrastructure-as-code — config lives alongside the app, not buried in a dashboard
